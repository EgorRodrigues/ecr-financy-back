Criação de Expense

Registro de saída financeira (pagamento esperado ou realizado).

Resumo

Endpoint
POST /expenses/ (expenses.py)
Persistência
repositories/expenses.py:create_expense
Tabela
expenses (models/expenses.py)
Campos centrais
amount, status, issue_date, account_id, contact_id, description.

Regras e efeitos

  • Se status=pago, calcula automaticamente total_paid = amount + interest + fine - discount.
  • Em update/delete existe uma trava: não permite alterar/remover expense vinculada a uma fatura de cartão (ver CreditCardInvoice.expense_id).

Referências: schemas/expenses.py:ExpenseCreate, repositories/expenses.py:create_expense, repositories/expenses.py:update_expense.

Variação: criação por parcelas

Endpoint
POST /expenses/installments (router)
Regra
installments_total precisa ser >= 2.
Resultado
Cria um grupo + N expenses (1 por parcela), com arredondamento na última parcela.

Detalhes (grupo e parcelas)

  • Cria 1 registro de grupo (ExpenseInstallmentGroup) e, em seguida, cria N expenses com installment_group_id, installment_number e installments_total.
  • Data de vencimento: first_due_date (se não vier, usa issue_date) e soma mês a mês; se o dia não existir no mês alvo, ajusta para o último dia do mês.
  • Valor por parcela: calcula per_installment com ROUND_HALF_UP em 2 casas; a última parcela recebe o “resto” para garantir que a soma feche exatamente em amount_total.
  • Se status=pago, calcula total_paid em cada parcela usando amount + interest + fine - discount.

Gestão do grupo

  • Listar grupos: GET /expenses/installment-groups (router) calcula agregados (quantidades por status, total pago e próximo vencimento).
  • Detalhar grupo: GET /expenses/installment-groups/{group_id} (router) retorna grupo + lista de parcelas ordenadas.
  • Atualizar grupo: PUT /expenses/installment-groups/{group_id} permite alterar description e active do grupo.
  • Cancelar grupo: POST /expenses/installment-groups/{group_id}/cancel marca como cancelado apenas as parcelas que ainda não estão pago e limpa total_paid dessas parcelas.
  • Desativar grupo: POST /expenses/installment-groups/{group_id}/deactivate seta active=false no grupo e em todas as parcelas.

  API->>Auth: Depends(verify_token)

  API->>Repo: executa ação
  alt cancel
    Repo->>DB: SELECT expenses WHERE installment_group_id=gid
    Repo->>DB: para cada parcela != pago, status=cancelado e total_paid=NULL
  else deactivate
    Repo->>DB: group.active=false
    Repo->>DB: para cada parcela, active=false
  end
  Repo->>DB: COMMIT
  API-->>Client: group + expenses
